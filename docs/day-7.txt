================================================================
APEX SPORTS - DAY 7: AI SERVICE & OLLAMA INTEGRATION
================================================================

Day 7 focused on integrating Generative AI into the platform to provide smart analysis and insights for sports data.

----------------------------------------------------------------
1. AI SERVICE ARCHITECTURE
----------------------------------------------------------------
The AI Service acts as a bridge between our platform and a local Large Language Model (LLM) engine:
- Express.js backend on Port 4003.
- Axios for communicating with the Ollama HTTP API.
- SSE (Server-Sent Events) for real-time streaming of AI responses.

----------------------------------------------------------------
2. OLLAMA INTEGRATION (LOCAL LLM)
----------------------------------------------------------------
We used Ollama as our inference engine, running the Llama 3.2 model.
- Why Ollama? It allows us to keep sports data private (local processing) and avoids high API costs from providers like OpenAI.
- Streaming: We implemented a custom stream handler to process chunks from Ollama and forward them to the client instantly.

----------------------------------------------------------------
3. PROMPT ENGINEERING (CONTEXT BUILDER)
----------------------------------------------------------------
AI is only as good as its context. We built a `contextBuilder.js` that automatically transforms raw JSON data into a narrative prompt.
- Input: User query + JSON data (Match scores, Player stats, CSPI comparisons).
- Logic: Formats the data into a structured system prompt that tells the AI exactly who it is (Apex AI) and what data it's looking at.
- Benefit: This prevents the AI from "hallucinating" scores and ensures it stays grounded in our platform's data.

----------------------------------------------------------------
4. STREAMING ENDPOINT (POST /api/ai/ask)
----------------------------------------------------------------
Unlike standard REST endpoints, the AI endpoint uses Server-Sent Events (SSE).
- Mechanism: The client sends a query, and the server keeps the connection open, sending tokens as they are generated by the model.
- UX: This allows the frontend to show a "typing" effect, significantly improving the perceived performance of the AI.

----------------------------------------------------------------
5. GATEWAY CONFIGURATION
----------------------------------------------------------------
The Gateway routes `/api/ai/**` to Port 4003. 
- Security: AI routes are protected by Firebase Auth middleware to prevent unauthorized usage of the local model resources.

----------------------------------------------------------------
INTERVIEW PREP - KEY QUESTIONS:
----------------------------------------------------------------
Q: Why use SSE instead of standard JSON response for AI?
A: AI generation is slow. If we wait for the entire 200-word response to be ready, the user might see a 5-10 second loading spinner. SSE allows us to show the first word in milliseconds, providing a much better user experience.

Q: How do you prevent the AI from talking about non-sports topics?
A: Through "System Prompting" in the Context Builder. We explicitly define the AI's persona as a "Specialized Sports Analyst" and limit its focus to the data provided in the context.

Q: What are the advantages of self-hosting a model like Llama 3.2 via Ollama?
A: 1. Zero latency over external networks. 2. Data privacy (no data leaves our server). 3. No per-token costs. 4. Complete control over the model version.
